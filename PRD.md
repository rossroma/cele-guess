# 产品需求文档 (PRD) - CeleGuess 明星猜猜看

## 1. 产品概述

### 1.1 产品名称
CeleGuess - 明星猜猜看

### 1.2 产品定位
一款轻量级的网页小游戏，通过展示明星照片让用户猜测明星姓名，适用于移动端和PC端。

### 1.3 目标用户
- 娱乐爱好者
- 明星粉丝
- 休闲游戏玩家

## 2. 功能需求

### 2.1 游戏开始页面

#### 2.1.1 页面布局
- **简洁设计原则**：页面元素精简，突出核心功能
- **响应式布局**：自适应移动端和PC端

#### 2.1.2 核心元素

**开始按钮**
- 尺寸：大按钮设计，便于点击
  - 移动端：最小宽度 280px，高度 60px
  - PC端：最小宽度 320px，高度 70px
- 位置：页面中央显著位置
- 样式：醒目的配色，明确的交互反馈

**筛选菜单**
- 入口：明显的筛选图标/按钮
- 交互：点击展开筛选面板

#### 2.1.3 筛选功能

**筛选维度**

1. **地区筛选**
   - 内地
   - 香港
   - 台湾
   - 日韩
   - 其他

2. **性别筛选**
   - 男
   - 女

3. **职业筛选**
   - 影视演员
   - 相声演员
   - 脱口秀演员

**筛选交互**
- 支持多选
- 显示已选择的筛选条件数量
- 提供"重置筛选"功能
- 提供"确认"按钮应用筛选

### 2.2 游戏页面

#### 2.2.1 照片展示
- **默认行为**：随机显示一张明星照片
- **照片尺寸**：
  - 移动端：占据屏幕主要区域，保持适当留白
  - PC端：居中展示，最大宽度 600px
- **照片加载**：显示加载动画，优化用户体验
- **照片质量**：清晰度适中，平衡加载速度和识别度

#### 2.2.2 答案按钮
- **位置**：照片下方居中
- **功能**：点击显示明星姓名
- **交互状态**：
  - 未点击：显示"查看答案"
  - 已点击：显示明星姓名
  - 样式变化：点击后按钮样式改变，表示已查看

#### 2.2.3 切换功能
- **滑动切换**（移动端）
  - 左滑：下一张
  - 右滑：上一张
- **按钮切换**（PC端）
  - 左右箭头按钮
  - 键盘左右方向键支持
- **切换行为**：
  - 切换后重置答案按钮状态
  - 随机显示新照片
  - 平滑过渡动画

### 2.3 数据存储

#### 2.3.1 数据格式（JSON）

```json
{
  "celebrities": [
    {
      "id": "001",
      "name": "明星姓名",
      "photo": "path/to/photo.jpg",
      "region": "内地",
      "gender": "男",
      "profession": "影视演员"
    }
  ]
}
```

#### 2.3.2 数据字段说明
- `id`: 唯一标识符
- `name`: 明星姓名
- `photo`: 照片路径（本地或CDN）
- `region`: 地区（内地/香港/台湾/日韩/其他）
- `gender`: 性别（男/女）
- `profession`: 职业（影视演员/相声演员/脱口秀演员）

## 3. 技术需求

### 3.1 响应式设计
- **移动端优先**：主要使用场景
- **断点设置**：
  - 移动端：< 768px
  - 平板：768px - 1024px
  - PC端：> 1024px

### 3.2 性能要求
- 页面加载时间：< 2秒
- 图片加载优化：懒加载、压缩
- 流畅的切换动画：60fps

### 3.3 浏览器兼容性
- Chrome（最新版及前2个版本）
- Safari（iOS 12+）
- Firefox（最新版及前2个版本）
- Edge（最新版）

## 4. 用户体验流程

### 4.1 首次进入
1. 加载游戏开始页面
2. 展示游戏标题和开始按钮
3. （可选）查看筛选菜单，设置筛选条件
4. 点击开始按钮进入游戏

### 4.2 游戏流程
1. 随机显示明星照片（根据筛选条件）
2. 用户思考或直接点击"查看答案"
3. 查看答案后，通过滑动/点击切换到下一张
4. 重复步骤2-3

### 4.3 返回首页
- 提供返回按钮
- 可重新设置筛选条件
- 开始新一轮游戏

## 5. UI/UX 设计要点

### 5.1 视觉设计
- **配色方案**：简洁明快，符合娱乐属性
- **字体**：清晰易读，适配中文
- **图标**：扁平化设计，符合现代审美

### 5.2 交互设计
- **触摸友好**：按钮尺寸符合触摸标准
- **反馈及时**：点击、滑动有明确反馈
- **动画流畅**：过渡自然，不卡顿

### 5.3 无障碍设计
- 适当的对比度
- 按钮和文字的可读性
- 支持放大缩小

## 6. 功能优先级

### P0（必须有）
- 游戏开始页面
- 开始按钮
- 照片展示
- 答案按钮
- 左右滑动/切换功能
- 基础筛选功能

### P1（应该有）
- 筛选面板UI优化
- 切换动画效果
- 加载状态提示

### P2（可以有）
- 游戏统计（查看了多少张）
- 收藏功能
- 分享功能

## 7. 数据准备

### 7.1 照片要求
- 格式：JPG/PNG
- 尺寸：建议 800x800px 或更大
- 文件大小：< 200KB（优化后）
- 内容：清晰、正面或半身照优先

### 7.2 数据量
- 初始版本：至少50位明星
- 覆盖各个筛选维度
- 均衡分布，避免某个分类数据过少

- 本地数据接入
- 移动端适配

### Phase 2: 优化迭代（1-2周）
- UI/UX优化
- 性能优化
- Bug修复
- 筛选功能完善

### Phase 3: 上线准备（1周）
- 测试
- 数据准备
- 部署

## 10. 成功指标

- 页面加载速度 < 2秒
- 移动端和PC端适配完美
- 用户能够流畅完成筛选、查看、切换等核心操作
- 无重大bug
- 游戏可玩性强，数据丰富
